# Métis UI Design System Overview

The Métis interface is designed as a precision intelligence instrument.

It must feel:

- authoritative
- analytical
- discreet
- premium
- calm under dense information load

It must not feel:

- playful
- generic SaaS
- neon cyberpunk
- trend-driven
- decorative for its own sake

## Core Model

The system is composed of three visible layers:

### 1. Environment
The environment establishes the intelligence workspace:
- dark background surfaces
- low-noise containers
- restrained borders and depth
- strong contrast for long analysis sessions

### 2. Authority Layer
The Authority Layer communicates confirmed intelligence and platform identity:
- Wisdom Golds
- primary focal entities
- structural emphasis
- premium branded moments

### 3. Signal Layer
The Signal Layer communicates analytical classification:
- anomalies
- financial activity
- relationship strength
- communications
- emerging patterns

Signal colors classify findings; they do not replace the core brand palette.

## Product Goals

The UI system must support:
- investigation-first workflows
- graph and timeline analysis
- evidence-backed reasoning
- dense but readable tables and cards
- report generation from live investigation states

## Output Modes

The UI system must support both:
- application surfaces
- report/export surfaces

The digital application may use darker branded presentation.  
Print-oriented output must remain clean, restrained, and efficient.
