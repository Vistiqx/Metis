# Metis Graph Visualization Specification

The intelligence graph is a **core analytical surface** in Metis.

Graph visualization must communicate:

• entity relationships  
• influence structures  
• investigative pathways  
• signal anomalies  

---

# Graph Composition

Graphs should feature:

• a central intelligence node  
• connected relationship nodes  
• directional relationship edges  

Graph visuals must prioritize **clarity over decoration**.

---

# Node Types

Nodes represent intelligence entities such as:

• people  
• organizations  
• assets  
• locations  
• cases  

Nodes must display:

• entity label  
• signal status  
• classification indicator  

---

# Signal Node Rendering

Signal nodes must use **signal colors only for intelligence state**.

Signal colors must **not change the base UI theme**.

Allowed signal types:

• anomaly  
• financial activity  
• confirmed intelligence  
• emerging pattern  
• core intelligence  

---

# Graph Edge Rendering

Edges represent relationships.

Edges should display:

• relationship strength  
• direction  
• classification  

Edge styling must remain **minimal** to prevent graph clutter.

---

# Interaction Behavior

Graphs must support:

• node inspection  
• relationship tracing  
• zoom and pan  
• entity focus mode  

These interactions allow analysts to explore investigative networks.
